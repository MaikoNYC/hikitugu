# hikituguチーム — Agent 用プロジェクト指示

**チーム名**: hikituguチーム  
このプロジェクトを開いた Agent およびチーム全員に共通で適用されます。

---

## このプロジェクトの目的（実現したい内容）

hikitugu資料を簡単に生成できる **MVP アプリの構築** に特化したチームです。

- **主要機能**: Googleカレンダー連携（予定に保存されている議事録も含む）、Slack連携、スプレッドシート連携から、**引き継ぎ資料を生成**する機能
- **担当範囲**: 要件定義 → 詳細設計 → 構築 → 運用・保守まで一貫して担当する
- **進め方**: 各工程で質疑応答をきちんと実施し、構築・リリース完了まで稼働する

---

## Agent が守ること（チーム共通）

- **言語**: 回答・コメント・ドキュメントは日本語で書く。
- **変更前の確認**: 破壊的変更や削除の前に、影響範囲を説明し必要なら確認を取る。
- **スコープ**: 上記「目的」の範囲内で作業する。他プロジェクトのコードは触らない。
- **工程**: 要件定義・設計・実装の各段階で、不明点は質疑応答で解消してから次に進む。

---

## 許可する操作

- ファイルの読み書き、ターミナル実行（ビルド・テスト・git など）は許可してよい。
- 本番データの直接変更・削除は行わない。必要な場合は手順を提案する。
